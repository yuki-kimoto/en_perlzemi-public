<h3> Grammar you should know </ h3>

Here's a list of grammars you should know that often appear in Perl.

<h4> Perl truth value </ h4>

<a href="/blog/20080323120628.html"> Perl boolean </a> has the following definition: There are four types of values ​​that Perl determines to be false: "undef", "''", "0", and "'0'". Everything else is true.

<h4> Determine if a defined value is defined </ h4>

Use the <a href="/blog/20100219126425.html"> defined function </a> to see if a value is defined.

<pre>
defined $ num;
</ pre>

<h4> Command line arguments </ h4>

Use the @ARGV variable to receive <a href="/blog/20080108132865.html"> command line arguments </a>.

<pre>
my ($ file, $ options) = @ARGV;
</ pre>

If you want to receive only one command line argument, you can write: Implicitly @ARGV is passed as an argument to shift.

<pre>
my $ file = shift;
</ pre>

<h4> Scalar context and list context </ h4>

In Perl, some functions have different return values ​​depending on the context. For example, the localtime function. This is a Perl-specific grammar called <a href="/blog/20080608121292.html"> context </a>.

<pre>
#Scalar context (get date / time as a character string)
my $ time_str = localtime ();

#List context (get each element of date and time as an array)
my @datetime = localtime ();
</ pre>

<h4> unless statement </ h4>

The <a href="/blog/20100125126425.html"> unless statement </a> expresses the negation of the if statement.

<pre>
unless (condition) {
  ...
}
</ pre>

<h4> Postfix if, Postfix unless </ h4>

In Perl you can put if or unless after a statement.

<pre>
# Postfix if
print $ num if $ num> 3;

# Postfix unless
die "error" unless $ num;
</ pre>

<h4> Postfix for </ h4>

In Perl you can put a for after a statement.

<pre>
# Postfix for
print $ _ for @nums;
</ pre>

The values ​​are sequentially assigned to "$ _".

<h4> Array slices and hash slices </ h4>

Use <a href="/blog/20080216120316.html"> array slice </a> and <a href="/blog/20080215120307.html"> hash slice </a> to get only the specified elements as an array I can do it.

<pre>
# Array slice
@select = @array [1, 4, 5];

# Hash slice
@select = @hash {'a','b','d'};
</ pre>

<h4> map function </ h4>

You can use the <a href="/blog/20100118126425.html"> map function </a> to transform each element of the array. The elements of @array are assigned to $ _ in order.

<pre>
@mapped = map {$ _ * 2} @array;
</ pre>

<h4> grep function </ h4>

You can use <a href="/blog/20100119126425.html"> grep function </a> to get only the elements that match the conditions as an array. The elements of @array are assigned to $ _ in order.

<pre>
@select = grep {$ _ = ~'cat'} @array;
</ pre>

<h4> List assignment </ h4>

<a href="/blog/20100308126967.html"> List </a> There is an assignment method called assignment.

<h4> Single return </ h4>

If you write return in a subroutine, <a href="/blog/20100220126425.html"> undef </a> will be returned in the case of scalar context, and an empty list () will be returned in the case of list context. .. If you want to use the return value to tell that an error has occurred in a subroutine, write return instead of return undef.

<pre>
sub name {
  my @args = @_;
  
  return;
}
</ pre>

There is no function to read everything from a file, so read all the lines into an array and connect them with the <a href="/blog/20081007122338.html"> join function </a>.

<pre>

</ pre>

This can be written in one line.

<pre>

<h4> Ternary operator </ h4>

<a href="/blog/20080325120646.html"> Ternary operator </a>. In the sample below, 1 is assigned to $ num if $ flg is true, and 2 is assigned to $ num if it is false.

<pre>
my $ num = $ flg? 1: 2;
</ pre>

<h4> || = </ h4>

Substitutes the rvalue if the lvalue is a false value. In the case of the sample below, 2 is assigned to $ num if $ num is a false value.

<pre>
$ num || = 2;
</ pre>

<h4> // = </ h4>

Defined-or operator. Substitutes the rvalue if the lvalue is undefined. In the sample below, 2 is assigned to $ num if $ num is undefined.

<pre>
$ num // = 2;
</ pre>

<pre>

my $ conf = do $ conf_file
  or die qq / Can't load config file "$ conf_file": $! $ @ /;
</ pre>

The contents of the configuration file.

<pre>

  name =>'Foo',
  number => 9

</ pre>

If the file is not read or parsed successfully, the undefined value will be returned. If the file name does not exist, $! Will be set, and if the Perl source code fails to load, $ @ will be set to the error content.
