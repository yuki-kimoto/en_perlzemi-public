    Installing Perl on Unix / Linux

    Perl is often already installed in Unix / Linux environments.

    Linux distributions include Ubuntu and CentOS > There is.

    Check if Perl is installed with the following command. 

    perl -v
</ pre>

You can start using Perl as soon as it exists.

If you don't have one, you can install the standard Perl for Unix / Linux distributions.

Alternatively, you can install Perl in your environment.


<h3> Install system Perl </ h3>

Execute the following command with root user privileges. Each distribution has a different command, so try running the command that suits your environment.

<h4> Installing Perl on Ubuntu </ h4>

How to install Perl on Ubuntu.

<pre>
#Install Perl on Ubuntu
sudo apt install perl
</ pre>

Please also refer to the following articles for installing Perl on Ubuntu.

<ul>
  <a href="https://ubuntu.perlclub.net/blog/20200616085351.html"> Install Perl on Ubuntu </a>
</ ul>

<h4> Installing Perl on Redhat Linux </ h4>

<pre>
#Install Perl on RedHat
yum install perl
</ pre>

<h4> Installing Perl on CentOS </ h4>

<pre>
#Install Perl on Cent OS
yum install perl
</ pre>

<h4> Installing Perl on Debian </ h4>

<pre>
#Install Perl on Debian
apt-get install perl
</ pre>

<h4> Installing Perl on FreeBSD </ h4>

<pre>
#Install Perl on FreeBSD
pkg_add -r perl
</ pre>

<h3> Installing Perl in the user environment </ h3>

If you want to install Perl in your own user environment, you can use a tool called perlbrew or plenv.

It doesn't require root privileges, so it's easy to install and doesn't pollute the OS environment. You can install and use the latest Perl.

If you're running Perl on a production web service, this method is the mainstream as of 2018.

Both perlbrew and plenv are similar tools for installing Perl in your environment. Please select as you like.
