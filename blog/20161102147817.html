    Perl special variables

    A list of Perl special variables. Special variables are global variables that are predefined in Perl. The variable name identifies its use. I will introduce only this from the special variables that I definitely want to remember.

    @ARGV --command line arguments

    command line argument is assigned to "@ARGV". 

    @ARGV
</ pre>

<h4> @ _ --Subroutine arguments </ h4>

The <a href="/blog/20180523152759.html"> argument </a> of the <a href="/blog/20101130129876.html"> subroutine </a> is assigned to "@ _".

<pre>
@_
</ pre>

<h4> $ 1, $ 2, $ 3, ... --Regex captured string </ h4>

For "$ 1", "$ 2" and "$ 3", use <a href="/blog/20100827127859.html"> regular expression </a> to <a href="/blog/20180524152759.html"> capture </a> The specified character string is assigned.

<pre>
$ 1;
$ 2;
$ 3;
</ pre>

<h4> $ _ --Default variable </ h4>

"$ _" Is called <a href="/blog/20180118151628.html"> default variable </a>. It's best to refrain from using default arguments yourself for readability, but some standard functions use default variables, so there are a few things to keep in mind.

At a minimum, <a href="/blog/20100118126425.html"> map function </a>, <a href="/blog/20100119126425.html"> grep function </a>, <a href="/blog" /20170521149524.html "> Postscript for </a> often uses" $ _ ", so keep in mind.

<b> map function </ b>

<pre>
map {$ _ => 1} @elements
</ pre>

<b> grep function </ b>

<pre>
grep {$ _ eq'foo'} @elements
</ pre>

<b> Postfix for </ b>

<pre>
$ _ for @elements;
</ pre>

<h4>% ENV --Getting and setting environment variables </ h4>

You can access <a href="/blog/20180510152716.html"> environment variables </a> with a special variable called <strong>% ENV </ strong>.

<pre>
% ENV
</ pre>

The environment variable name is the key and you can refer to the value with $ ENV {'env_name'}. Environment variables are like variables that the OS has. Environment variables are copied into the memory of the process space when the OS starts the process. OS environment variables cannot be changed from the process.
Environment variables can be set freely, but they are valid only in that process. When a process launches a child process, the environment variables are copied, as if the OS launched the process.

<h4> $ 0 --Get program name </ h4>

Use the <strong> special variable "$ 0" </ strong> to <a href="/blog/20180509152716.html"> get the program name </a>.

<pre>
$ 0;
</ pre>

Gets the name when the program was started. In other words, if you specify a program with an absolute path, you can get the program name with the absolute path name from the current directory, and if you specify it with a relative path, you can get the program name with the relative path name.

<h4> @INC-List of module search paths </ h4>

<a href="/blog/20180508152716.html"> List of module search paths </a> is assigned to a special variable called <strong> @ INC </ strong>.

<pre>
@INC
</ pre>

The module search path is the directory that Perl searches to load the module. Modules placed in directories not included in @INC cannot be loaded.

<b> Use the lib module to add a module search path </ b>

<pre>
use lib'./lib';
</ pre>

You can add the search path to @INC at compile time by using the <a href="/blog/20100525127859.html"> lib </a> module.

<b> Add the module search path directly to the beginning of @INC </ b>

<pre>
unshift @ INC,'./lib2';
</ pre>

Add it directly to the beginning with <a href="/blog/20100105126425.html"> unshift function </a>. It is better to use use lib than this way. The advantage of using use lib is that it adds a search path at compile time, makes it more readable, and adds an architectural path as well.

<h4>% INC --Check loaded modules </ h4>

To find out <a href="/blog/20180529152759.html"> loaded modules </a>, look at the variable <strong>% INC </ strong>.

<pre>
% INC
</ pre>

The% INC hash is assigned a combination of module name and module file name. You can see the module name and which file you actually imported.

It has nothing to do with @INC, which stores the module's search path. @INC and% INC are completely different variables. If you want to know only the module name, use the <a href="/blog/20080210120265.html"> keys function </a> to retrieve only the keys.

<h3> Other special variables </ h3>

<h4> $$ --Get process ID </ h4>

Use the special variable "<strong> $$ </ strong>" to <a href="/blog/20180520152690.html"> get the process ID </a>.

<pre>
$$
</ pre>

The process ID is the unique knowledge of the process by the OS.To separate it, it is an identifier assigned when a process is started.

<h4> $ ^ T --Get the process start time </ h4>

<a href="/blog/20180526152759.html"> To get the process start time </a>, use the special variable "<strong> $ ^ T </ strong>".

<pre>
$ ^ T
</ pre>

The time is obtained in epoch seconds (seconds from 00:00:00 on January 1, 1970).

<h4> $? --Child process exit status </ h4>

Use "$?" To get the <a href="/blog/20180525152759.html"> exit status of the child process </a>.

<pre>
$?
</ pre>

If wait waits for the child process to terminate, $? Contains multiple values, including the exit status of the child process. $? Is also set when a child process is executed using the system function.
