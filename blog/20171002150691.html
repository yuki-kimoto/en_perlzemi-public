    You can use the opdndir function to open a directory and get a directory handle. 

    opendir directory handle directory name
</ pre>

Specify an undefined variable for the directory handle. Use the <b> opendir </ b> function as follows:

<pre>
# Directory name
my $ dir ='study';

#Open directory
opendir my $ dh, $ dir
  or die "Can't open directory $ dir: $!"
</ pre>

An undefined variable is specified in the first argument, but <a href="/blog/20161107147852.html"> variable declaration by my </a> is performed at the same time. The second argument specifies the directory name.

If the directory opening fails, a false message will be returned, so you need to handle the error. Use <a href="/blog/20100816136168.html"> or operator </a> to describe the processing when a failure occurs.

<a href="/blog/20100212126425.html"> die function </a> raises an exception and terminates the program.

The <a href="/blog/20161102147817.html"> special variable "$!" </a> is assigned the reason why the directory open failed, so be sure to include it in the error message.

<h4> How to close the directory handle? </ H4>

The directory handle is automatically released when the <a href="/blog/20110128130016.html"> scope </a> is finished, so there is no need to explicitly close it. This is because the destructor runs when there are no more references to the directory handle.

<pre>
{
  # Directory name
  my $ dir ='study';

  #Open directory
  opendir my $ dh, $ dir
    or die "Can't open directory $ dir: $!"
}

#The directory handle is automatically closed when the scope ends
</ pre>

If you want to explicitly close the directory handle, use the <a href="/blog/20171001150694.html"> closedir function </a>.

<pre>
closedir $ dh;
</ pre>
