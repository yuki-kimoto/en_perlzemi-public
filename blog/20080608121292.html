    Understand Perl context

    Perl has a concept called context. Context means context, and there are the following two contexts.

    • Scalar context
    • List context

      The context evaluated as a scalar is called a scalar context, and the context evaluated as a list is called a list context.

      As an example, the localtime function returns a string that displays the date / time in the scalar context, and the date / time information in the list context. Is listed and returned. 

      #Scalar context
$ date_string = localtime;

#List context
@date_info = localtime;
</ pre>

To give another example, an array returns the number of arrays in a scalar context and a list of elements in a list context.

<pre>
#Scalar context
$ count = @values;

#List context
@elements = @values;
</ pre>

<h4> Scalar context </ h4>

Here are some places that are evaluated as scalar contexts.

<b> Assignment to scalar variables </ b>

Assignment to <a href="/blog/20161107147852.html"> scalar variable </a> is a scalar context. When the array is evaluated in scalar context, it returns the length of the array.

<pre>
my $ num = @values;
</ pre>

<b> Comparison operator term </ b>

The left and right terms of <a href="/blog/20080318120585.html"> comparison operator </a> are scalar contexts.

<pre>
$ x <@values
$ x == @values
</ pre>

I often see it in the loop of <a href="/blog/20170521149524.html"> for statement </a>.

<pre>
for (my $ i = 0; $ i <@nums; $ i ++) {
  ...
}
</ pre>

<b> if condition part </ b>

The conditional part of <a href="/blog/20161023147731.html"> if </a> is the scalar context. In the following, @values ​​is evaluated in scalar context, so it returns the number of arrays.

<pre>
if (@values) {
  ...
}
</ pre>

<h4> List context </ h4>

Here's where it's evaluated as a list context.

<b> Assignment to an array </ b>

Assignment to <a href="/blog/20161012147627.html"> array </a> is a list context.

<pre>
my @ values2 = @values;
</ pre>

<b> List </ b>

Inside the <a href="/blog/20100308126967.html"> list </a> is the list context.

<pre>
(@values);
</ pre>

<b> Subroutine arguments </ b>

The argument of <a href="/blog/20101130129876.html"> subroutine </a> is the list context.

<pre>
func (@values);
</ pre>

<h3> Sample </ h3>

This is a sample to understand the scalar context and list context.

<pre>
use strict;
use warnings;

#Scalar context and list context

print "1: Know the meaning of the context with the localtime function. \ N";
my $ scalar;
my @list;

#Since the left side is scalar, it is for scalar
The return value of # is returned.
$ scalar = localtime;
                      
#The left side is an array, so it's for a list
The return value of # is returned.
@list = localtime;

print "formatted date \ n in scalar context",
  "'$ scalar' \ n \ n";

print "In list context, a list of date and time values ​​\ n".
  join (',', @list). "\ n \ n";
</ pre>

Perl functions can return a scalar return where a scalar is required and a list return where a list is required.

The distinction between a scalar context and a list context is naturally remembered as you touch Perl. Here is an example. If the left side is a scalar, it is a scalar context, and if the left side is an array, it is a list context.
