    or operator --OR with low priority

    or is one of Perl's logical operators and represents a logical sum. It has the same meaning as ||, but or has a lower operator priority and is generally used between sentences. 

    Sentence 1 or Sentence 2
</ pre>

With this description, statement 2 is executed when statement 1 returns false.

Use <h3> or as a conditional branch </ h3>

Taking advantage of these properties, or is often used for conditional branching. For example, if the <a href="/blog/20080718121643.html"> open function </a> fails, write as follows.

<pre>
open my $ fh,'<', $ file
  or die qq / Can't open file "$ file": $! /;
</ pre>

If the open function succeeds, true is returned and the right-hand side of or is not executed. If the open function fails, undef is returned, so <a href="/blog/20100212126425.html"> die </a> on the right side is executed and the program ends.

As another example, it is also used with the system function. The system function returns 0 on success, so write:

<pre>
system (@cmd) == 0
  or die "Can't execute command @cmd";
</ pre>

If you want to write the main processing first and write supplementary error processing later, you can use conditional branching with or.

You can use "and" for conditional branching, but "or" is recommended because it is difficult to read.

<pre>
Example rewritten with # and
system (@cmd)
  and die "Can't execute command @cmd";
</ pre>
