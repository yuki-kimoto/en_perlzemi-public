<h4> sqrt </ h4>

To find the positive square root, use the <a href="/blog/20080630121483.html"> sqrt function </a>.

<pre>
$ ret = sqrt $ num;
</ pre>

sqrt (4) will be 2.

<h4> int </ h4>

Use the <a href="/blog/20080923122214.html"> int function </a> to retrieve the integer part of a number that includes a decimal point.

<pre>
$ ret = int $ num;
</ pre>

int (1.5) becomes 1 and int (-1.5) becomes -1.

<h4> ceil (POSIX) </ h4>

To round up the decimal point, use the POSIX module's <a href="/blog/20080924122227.html"> ceil function </a>.

<pre>
use POSIX'ceil';
$ ret = ceil $ num;
</ pre>

<h4> floor (POSIX) </ h4>

To round down the decimal point, use the <a href="/blog/20090430122227.html"> floor function </a> of the POSIX module.

<pre>
use POSIX'floor';
$ ret = floor $ num;
</ pre>

<h4> rand </ h4>

Use the <a href="/blog/20080906122079.html"> rand function </a> to generate random numbers. A number smaller than the number specified by the argument is randomly generated.

<pre>
$ ret = rand $ num;
</ pre>

<h4> looks_like_number (Scalar :: Util) </ h4>

<h3> 2 String manipulation </ h3>

<h4> join </ h4>

Use the <a href="/blog/20081007122338.html"> join function </a> to concatenate strings by specifying a delimiter.

<pre>
$ ret = join (',', @array);
</ pre>

The first argument is the delimiter. In this example, a comma is specified as the delimiter. The character strings given in the second and subsequent arguments are concatenated with a delimiter.

<h4> length </ h4>

To get the length of the string, use the <a href="/blog/20080926122245.html"> length function </a>.

<pre>
$ ret = length $ str;
</ pre>

<h4> substr </ h4>

Use the <a href="/blog/20080224120386.html"> substr function </a> to extract and replace the characters at the specified position.

[A] Extract the character at the specified position

<pre>
$ ret = substr ($ str, 2, 4);
</ pre>

The first argument is the target string. The second argument is the start position of the string you want to extract. The third argument is the length of the string you want to extract.

[B] Replace the character string at the specified position

<pre>
substr ($ str, 0, 2, $ replace);
</ pre>

To replace the character string at the specified position, specify the replaced character string in the 4th argument. In this example, the first two characters of $ str are replaced by $ replace.

<h4> index </ h4>

Use the <a href="/blog/20080225120395.html"> index function </a> to search for a string.

<pre>
$ ret = index ($ str, $ search);
</ pre>

In the first argument, specify the character string to be searched, and in the second argument, specify the character string you want to search. The position where the character string is found is returned as the return value. If not found, -1 will be returned.

<pre>
$ ret = index ($ str, $ search, $ pos);
</ pre>

You can specify the search start position in the third argument.

<h4> rindex </ h4>

Use the <a href="/blog/20100321126967.html"> rindex function </a> to search for a string from the end of the string.

<pre>
$ ret = rindex ($ str, $ search);
</ pre>

<h4> split </ h4>

Use the <a href="/blog/20080301120434.html"> split function </a> to split the string by the delimiter.

<pre>
@ret = split (/, /, $ str);
</ pre>

Specify the delimiter as a regular expression in the first argument. In the second argument, specify the character string you want to split. The return value is an array of split strings.

<h4> reverse </ h4>

Use the <a href="/blog/20080227120413.html"> reverse function </a> to reverse the order of the strings.

<pre>
$ ret = reverse $ str;
</ pre>

Also, when used on an array, the reverse function reverses the order of the array.

<h4> uc </ h4>

Use the <a href="/blog/20080903122057.html"> uc function </a> to convert the string to uppercase.

<pre>
$ ret = uc $ str;
</ pre>

<h4> ucfirst </ h4>

<pre>

</ pre>

<h4> lc </ h4>

Use the <a href="/blog/20100326127089.html"> lc function </a> to convert the string to lowercase.

<pre>
$ ret = lc $ str;

<h4> lcfirst </ h4>

Use the <a href="/blog/20100327127089.html"> lcfirst function </a> to convert only the beginning of a string to lowercase.

<pre>

</ pre>

<h4> sprintf </ h4>

 Use the <a href="/blog/20081001122296.html"> sprintf function </a> to create a formatted string.

<pre>
$ ret = sprintf ($ format, @strs);
</ pre>

The first argument is a string that represents the format. The second and subsequent arguments are the character strings embedded in the format specifier part.

<span style = "font-weight: bold;"> [A] Formatting sample </ span>

Interpret $ num as an integer and format it with 8 digits. Parts less than 8 digits are padded with 0s.

<pre>
$ ret = sprintf ("% 08d", $ num);
</ pre>

Interpret $ num as a floating point and round it to two decimal places.

<pre>
$ ret = sprintf ("% .2f", $ num);
</ pre>

<span style = "font-weight: bold;"> [B] List of format specifiers </ span>
<pre>
% c Convert numbers to characters corresponding to ASCII code
Interpret the% s argument as a string
Interpret the% d argument as a signed integer
Interpret the% u argument as an unsigned integer
% o Convert positive numbers to octal strings
% x Convert positive numbers to hexadecimal strings (lowercase notation)
% X Convert positive numbers to hexadecimal strings (uppercase)
% b Convert positive numbers to binary strings
Interpret the% f argument as a floating point
Interpret% e argument as floating point (formatted in exponential notation (lowercase))
Interpret% E argument as floating point (formatted in exponential notation (uppercase))
Interpret% g argument as floating point (formatted in exponential notation (lowercase), trailing 0s removed)
Interpret the% G argument as a floating point number (finger)Formatted in number notation (uppercase). The trailing 0 is removed)
Translated to the in-memory address of the variable given in the% p argument
</ pre>

<h4> chomp </ h4>

<pre>

<h4> Quart operator "q" </ h4>

If you want to use single quotes without escaping them, use <a href="/blog/20100304126967.html"> quote operator </a>.

<pre>
q (aaa'bbb)
</ pre>

The function is the same as when enclosed in single quotes. No variable expansion is done.

<pre>
# Same as this
'aaa \'bbb'
</ pre>

<h4> Double quote operator "qq" </ h4>

If you want to use double quotes without escaping them, use <a href="/blog/20100305126967.html"> double quote operator </a>.

<pre>
qq (aaa "bbb)
</ pre>

The function is the same as when enclosed in double quotes. Variable expansion is performed.

<pre>
# Same as this
"aaa \" bbb "
</ pre>

</ pre>

<h4> unshift </ h4>

Use the <a href="/blog/20100105126425.html"> unshift function </a> to add an element to the beginning of the array.

<pre>
unshift (@array, @items);
</ pre>

<h4> pop </ h4>

Use the <a href="/blog/20100107126425.html"> pop function </a> to retrieve the last element of the array.

<pre>
$ ret = pop @array;
</ pre>

Use the <a href="/blog/20100106126425.html"> push function </a> to add an element to the end of the array.

<pre>

</ pre>

<h4> splice </ h4>

Use the <a href="/blog/20080127120141.html"> splice function </a> to work with multiple elements on an array.

<span style = "font-weight: bold;"> [A] Extract multiple elements </ span>
You can retrieve multiple elements.

<pre>
@ret = splice (@array, $ pos, $ size)
</ pre>

Specify the target array in the first argument, the start position in the second argument, and the size you want to extract in the third argument.

<span style = "font-weight: bold;"> [B] Replace multiple elements </ span>
You can replace multiple elements.

<pre>
splice (@array, $ pos, $ size, @items);
</ pre>

Specify the array you want to replace with the 4th argument.

<h4> Array slice </ h4>

Use <a href="/blog/20080216120316.html"> Array Slice </a> to get multiple elements from an array.

<pre>
@ret = @array [0, 1, 3];
</ pre>
In this example, the 0th, 1st, and 3rd elements of @array are assigned to @ret.

<h4> grep </ h4>

If you want to retrieve only the elements that match the conditions, use the <a href="/blog/20100119126425.html"> grep function </a>.

<pre>
@ret = grep {condition} @array;
</ pre>

This is a sample to extract a value larger than 3.

<pre>
@ret = grep {$ _> 3} @array;
</ pre>

The elements of the array are sequentially assigned to $ _. Only those that satisfy $ _> 3 will be assigned to @ret.

<h4> map </ h4>

Use the <a href="/blog/20100118126425.html"> map function </a> to process all the elements of the array.

<pre>
@ret = map {element manipulation} @array;
</ pre>

This is a sample that doubles all the elements.

<pre>
@ret = map {$ _ * 2} @array;
</ pre>

The elements of the array are sequentially assigned to $ _. The result of $ _ * 2 is assigned to @ret.

<h4> sort </ h4>

Use the <a href="/blog/20100110126528.html"> sort function </a> to sort the elements of the array.

<pre>
@ret = sort {sorting conditions} @array;
</ pre>

<span style = "font-weight: bold;"> [A] Compare numerically and sort in ascending order </ span>
This is a sample that compares numerically and sorts in ascending order.

<pre>
@ret = sort {$ a <=> $ b} @array;
</ pre>

Use <=> to compare numerically. To sort in ascending order, use $ a <=> $ b.

<span style = "font-weight: bold;"> [B] Compare numerically and sort in descending order </ span>
This is a sample that compares numerically and sorts in descending order.

<pre>
@ret = sort {$ b <=> $ a} @array;
</ pre>

Use <=> to compare numerically. To sort in descending order, use $ b <=> $ a.

<span style = "font-weight: bold;"> [C] Compare in dictionary order and sort in ascending order </ span>
This is a sample that compares in dictionary order and sorts in ascending order.

<pre>
@ret = sort {$ a cmp $ b} @array;
</ pre>

Use cmp to compare in lexicographical order. If you want to sort in ascending order, use $ a cmp $ b.

<span style = "font-weight: bold;"> [D] Compare in dictionary order and sort in descending order </ span>
This is a sample that compares in dictionary order and sorts in descending order.

<pre>
@ret = sort {$ b cmp $ a} @array;
</ pre>

Use cmp to compare in lexicographical order. If you want to sort in descending order, use $ b cmp $ a.

It is necessary to handle the error when the file open fails.

for my $ diag (@ $ errors) {
  my ($ file, $ message) = each% $ diag;
  warn "problem unlinking $ file: $ message \ n";
}
</ pre>

In the first argument, specify the directory you want to delete. You can specify options for the last argument. In this example, I am trying to get the content of the error.

<h3> 7 Time / Time </ h3>

<h4> time </ h4>

Use the <a href="/blog/20081008122347.html"> time function </a> to get the elapsed seconds from the epoch to the present.

<pre>
$ ret = time;
</ pre>

The epoch time is January 1, 1970.

<h4> localtime </ h4>

Use the <a href="/blog/20081009122357.html"> localtime function </a> to get the current date and time.

<pre>
($ sec, $ min, $ hour, $ mday, $ mon, $ year, $ wday) = localtime;
$ year + = 1900;
$ mon + = 1;
</ pre>

From the beginning, the order is "seconds, minutes, hours, days, months, years, days of the week". The year is an elapsed year from 1900, so you need to add 1900 to the year you acquired it. Months start at 0, so you need to add 1 to get the actual month.

<h4> timelocal (Time :: Local) </ h4>

To get the elapsed seconds since the epoch from the date and time information, use the <a href="/blog/20081014122416.html"> Time :: Local module's timelocal function </a>.

<pre>
use Time :: Local'timelocal';
$ ret = timelocal ($ sec, $ min, $ hour, $ mday, $ mon, $ year);
</ pre>

<h4> sleep </ h4>

Use the <a href="/blog/20170522149546.html"> sleep function </a> to sleep. The sleep time is specified in seconds.

<pre>
sleep $ sec;
</ pre>

When it sleeps, the program will stop during that time.

<h4> usleep (Time :: HiRes) </ h4>

To sleep in microseconds, use the usleep function of the <a href="/blog/20180309152509.html"> Time :: HiRes </a> module.

<pre>
use Time :: HiRes'usleep';
usleep $ microsec;
</ pre>

<h4> File :: Spec </ h4>

You can use <a href="/blog/20100412127089.html"> File :: Spec module </a> to create a file name according to the OS.

<pre>

my $ file_name = File :: Spec-> catfile ('dir','file.txt');
</ pre>

On Windows operating systems, $ file_name is dir \ file.txt, and on Unix operating systems, the file name is dir / file.txt.

<h4> FindBin </ h4>

</ pre>

$ FindBin :: Bin contains the name of the directory where the script you are running is located.

<h4> Get :: Options </ h4>

If you want to receive options with <a href="/blog/20080108132865.html"> command line arguments </a>, use <a href="/blog/20100514127696.html"> Get :: Options module </a> To use. For example, suppose you want to receive the following options:

<pre>

<pre>

  'length = i'=> \ $ length,

);
</ pre>
