    Understand Perl scope

    Get a better understanding of Perl scope. This article is written on the assumption that the strict pragma is enabled, so please write the following at the beginning of the source code as usual. 

    use strict;
use warnings;
</ pre>

<h3> Create scope </ h3>

Perl scope starts with "{" and ends with "}".

<pre>
{
  #Scope
}
</ pre>

<h3> Meaning of scope </ h3>

<h4> Lexical variables </ h4>

Lexical variables declared within a scope (variables declared with my) can only be referenced within that scope. Also, the value contained in the variable is released when the scope ends.

<pre>
{
  my $ num = 1;

  # Can be referenced
  print $ num;
}

# Cannot be referenced at this position
print $ num;
</ pre>

<h4> Package variables </ h4>

Package variables declared inside a scope (variables declared with our) can be referenced by partial name within that scope. You must use a fully qualified name to reference it from outside the scope.

<pre>
package Foo;

{
  our $ NUM = 1;

  # Can be referenced by part name
  print $ NUM;
}

# Cannot be referenced by partial name
print $ NUM;

# Can be referenced by fully qualified name
print $ Foo :: NUM;
</ pre>

Keep in mind that once a package variable is created, it exists to the end of the program and can be referenced by its fully qualified name from anywhere in the program.

<h4> Package variables modified locally </ h4>

You can use <a href="/blog/20080309120507.html"> local </a> to temporarily change a package variable. This changed value is reverted at the end of the scope.

<pre>
package Foo;

our $ NUM = 1;

{
  local $ NUM = 2;

  # 2 is output
  print $ NUM;
}

# Return to 1 at this position
print $ NUM;
</ pre>

Use local only for temporary changes to the value of package variables, and don't use it for variable declarations as it used to be.

<h4> Subroutine </ h4>

Perl subroutines are scope-independent.

<pre>
{
  sub foo {
    ...
  }

  # Can be called
  foo ();
}

# Can be called
foo ();
</ pre>

When a subroutine is defined, it is registered in what is called a symbol table and can be called from any position in the program.

If you want the subroutine to have scope, assign an anonymous subroutine (which is the subroutine's reference) to a lexical variable.

<pre>
{
  my $ foo = sub {...};

  # Can be called
  $ foo-> ();
}

# Cannot be called from this position
$ foo-> ();
</ pre>

At the end of the scope, the subroutine reference assigned to $ foo is released. Note that the subroutine itself is not freed. sub {...} is parsed and placed in memory at compile time.

<h3> Scope in if and while statements </ h3>

The "{}" used in if statements and <a href="/blog/20100123126425.html"> while statements </a> is also a scope. In Perl, the scope is "{}", which is used in all kinds of syntax. It's consistent in this regard and is very straightforward compared to other languages.

<pre>
if (condition) {
  #Scope
}

while (condition) {
  #Scope
}

eval {
  #Scope
};

sub {
  #Scope
}
</ pre>

(Reference) <a href="/blog/20100312126967.html"> eval </a>

Files also have one scope.

<pre>
use strict;
use warnings;

# process
</ pre>

The above can be thought of as the beginning and end of the file surrounded by "{" and "}" as shown below.

<pre>
{
  use strict;
  use warnings;

  # process
}
</ pre>

<h3> Lexical variables declared at special positions </ h3>

There are some cases where you have a scope, though it is not declared inside the scope.

<h4> if condition part </ h4>

The lexical variable declared in the conditional part of if has the scope of if.

<pre>
if (my $ num = 1) {
  # Can be referenced
  print $ num;
}

# Cannot be referenced at this position
print $ num;
</ pre>

Since there is no scope for the postfixed if in the first place, it does not have a scope.

<pre>
Statement if my $ num = 1;

# Can be referenced
print $ num;
</ pre>

The same is true for unless statements and <a href="/blog/20100123126425.html"> while statements </a>.

<pre>
# unless
unless (my $ num = 1) {
  # Can be referenced
  print $ num;
}

# Cannot be referenced
print $ num;
</ pre>

<pre>
# while
while (my $ num = 1) {
  # Can be referenced
  print $ num;
}

# Cannot be referenced
print $ num;
</ pre>

<h4> for condition part </ h4>

The lexical variable declared in the condition part of for has the scope of for.

<pre>
for (my $ i = 0; $ i <@nums; $ i ++) {
  # Can be referenced
  print $ i;
}

# Cannot be referenced
print $ i;
</ pre>

<h4> variable assignment part of for </ h4>

The lexical variable declared in the variable assignment part of for has the scope of for.

<pre>
for my $ num (@nums) {
  # Can be referenced
  print $ num;
}

# Cannot be referenced
print $ num;
</ pre>

<h4> At the end </ h4>

If you keep this in mind, you won't have any problems with Perl scope. You'll find that Perl scope is intuitive and straightforward. This has been very helpful in making the structure of Perl programs easier to see.
