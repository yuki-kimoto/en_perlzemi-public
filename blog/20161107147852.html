<h4> Saving and restoring values ​​with "local" </ h4>

You can use "local" to temporarily save a package variable and restore it at the end of scope.

<pre>
our $ NUM = 5;
{
  local $ NUM = 3;
}
# Here, "$ NUM" returns to "5" which is the "original value"
</ pre>

<h3> Perl variable type </ h3>

Perl has three "variable types": "scalar variables", "array variables", and "hash variables". Please note that it does not indicate the type of a number like "int type" and "double type" like C language and Java.

<h4> "Scalar variable"-stores one value </ h4>

A "scalar variable" can contain only one value within a variable. A scalar variable is represented by prefixing the variable name with "$".

<b> Typical scalar value </ b>

Typical scalar values ​​are as follows.

<ul class = "check-list">
<ul>
  <li>
    String
  </ li>
  <li>
    Numerical value
  </ li>
  <li>
    reference
  </ li>
  <li>
    Undefined value (<a href="/blog/20100220126425.html"> undef </a>)
  </ li>
</ ul>
</ ul>

<b> Sample of assignment of scalar value to scalar variable </ b>

<pre>
my $ scalar;
$ scalar = "one"; #string
$ scalar = 2; #number
$ scalar = [3, 4, 5]; # Reference to array
$ scalar = {x => 6}; # Reference to hash
$ scalar = sub {return 7}; #Reference to subroutine
$ scalar = undef; # undefined value
</ pre>

<h4> "Array variable" --stores multiple values ​​</ h4>

An "array variable" can contain multiple values. Only scalar variables can be included in the "elements" of an array variable.

An array variable is represented by prefixing the variable with "@". Only "list ()" can be assigned to an array variable.

<pre>
# Array variables can have scalars as elements
my @elements = (
  "one",
  2,
  [3, 4, 5],
  {x => 6, y => 7},
  sub {return 8},
  undef
)
</ pre>

<b> Array nesting results in array concatenation </ b>

In Perl, array nesting is an array concatenation.

<pre>
my @list_in_list = ((1, 2), (3, 4));
# It will be the same as this.
my @list = (1, 2, 3, 4);
</ pre>

<h4> "Hash variable"-stores key / value pairs </ h4>

The hash has the following characteristics.

Includes multiple values ​​that are a set of "key and value". Only scalar variables can be included in the "elements" of a hash variable.

A hash variable is represented by prefixing the variable with "%". Only "list ()" can be assigned to a hash variable.

<pre>
my% scores = (math => 70, engligh => 80);
</ pre>

<b> Array-type to hash-type conversion </ b>

Both array variables and hash variables have a list as their content, so they can be converted to each other.

<pre>
@array =% hash;
% hash = @array;
</ pre>

<h3> Variables and scope </ h3>

Another thing to know about variables is their <b> scope </ b> and <b> lifetime </ b>. Scope is a term that describes where a variable is visible. Lifespan is the length of time a variable stores a value.

For example, for a lexical variable declared with my, the scope is from "{" to "}".

<pre>
{
  # Declare here
  my $ num = 3;
}
# I can't see from here
</ pre>

The value saved in "$ num" disappears from the memory when the scope by "{...}" ends, and the variable itself called $ num disappears.
