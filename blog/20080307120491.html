my - Declaration of lexical variables

You can declare lexical variables using my . 

# Declaration of lexical variables
my $num;
my @nums;
my %scores;

A lexical variable is a local variable in C language.

Perl has the keyword local, but it doesn't actually create a local variable. The local keyword has the ability to temporarily save and restore package variables.

Variable declaration by my is a feature added in Perl 5.6. This is the local variable itself in other languages, but since the keyword local already exists, it may have been named lexical variable so as not to be confusing.

Think of lexical variables in Perl as local variables in other languages.

What is a variable declaration?

The variable declaration declares that we will use variables from now on. Declare new variables using "my". 

# Declaration of scalar variables
my $num;

In addition to scalar variables, you can also declare array variables and hash variables. I can do it. 

# Declaration of array variables
my @nums;

# Hash variable declaration
my %score;

Variable initialization

"Variable initialization" can be performed at the same time as the variable declaration. 

# Initialization of scalar variables
my $num = 3;

# Initialization of array variables
my @nums = (1, 2, 3);

# Initialization of hash variables
my %score = (math => 80, english => 70);

If initialization is not performed, undefined value will be assigned for scalar variables. For array variables and hash variables, empty list is assigned.

What are lexical variables?

I wrote that you can declare a "lexical variable" using my, but what exactly is a lexical variable? A lexical variable is a variable that has a lexical scope.

Lexical variables have reached the end of their life and are freed from memory at the end of scope. 

# Start of scope
{
  my $num = 2;
}
# "$num" is not available here.

Difference in variable declaration by "my", "our" and "local"

There are three Perl variable declarations: "my", "our", and "local".

The simple difference is that "my" declares a "lexical variable", "our" declares a "package variable", and "local" declares a package variable and saves the value until the end of scope. I have.

Avoid using "our" or "local" when declaring variables in Perl as much as possible. In modern Perl, you can do complete programming with just a variable declaration with "my".
